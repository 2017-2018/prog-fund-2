Objectives

On completion of this lab you should be able to understand the ideas behind abstraction, sucessfully implement abstraction in the Network system and run the debugger on java code.

Network-V5

In this step, you will create version 5 of the Network system (our menu driven messaging system). In this version, you will use abstraction in the inheritance hierarchy.

Abstraction in Network-V5

  • In Eclipse, create a new project called Network-V5.

  • Copy the following classes into this new project:

    • PhotoPost
    • MessagePost
    • Post
    • NewsFeed
    • Driver

  • Make the Post class abstract and add an abstract method called displayExcerpt() to it. The return type of this method should be a String.

  • In Post's subclasses, provide the concrete implementation for this method.

  • Message Post code:

    public String displayExtract()
    {
        return "Message extract "+ message.substring(0,10) + "....";
    }
  • PhotoPost code:
    public String displayExtract()
    {
        return "Photo caption: " + caption.substring(0,10) + "....";
    }

  • Save your code.

  • Note that this code does not "do" anything as we have not called the displayExtract method from anywhere. This is just a very short lab to illustrate the abstract class and method concept.

Debugging

In this step, you will examine code using the debugger. This code has a logic error.

Debugging 'Debug Largest'

  • Import the following project into Eclipse

    • DebugLargest

  • Run the program (by running the Driver). The code is calculating the largest number. The correct answer is 5.

  • What is the answer given? To see why the answer is incorrect, we can no step through the code and examin the value of each of the variables at each point in the program.

  • To do this, place a breakpoint in the Largest class. Step through the code.

  • Experiment with this code. Ensure that you are familiar with the debugger.
